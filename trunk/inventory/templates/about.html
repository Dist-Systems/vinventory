This inventory system was designed to keep track of the various physical and virtual systems (regardless of their current use) that are under the care of an organization. The system was made using the Pythonic framework Django.

The system expects to be populated by some csv files (assumed to be PowerShell exports from VSphere) which are arranged as follows:

  • DataStore.csv
    Name,CapacityMB,FreeSpaceMB,FileSystemVersion
  • VMHost.csv
    Name,Manufacturer,Model,NumCpu,CpuTotalMhz,CpuUsageMhz,ProcessorType
  • VirtualMachine.csv
    Name,PowerState,NumCpu,MemoryMB,Notes,VMHost,Datastore
  • IpAddress.csv
    Name,ip

These files are assumed to be kept in the data subfolder of django django project. The script that reads the data files is part of the svn project, and is located in the scripts subfolder of the django project. Since the vsphere data is 'read-only': every time the script runs, it resets the tables associated with vsphere objects and then repopulates them entirely.
